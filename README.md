# Restaurant Management System – SQL Project

## Project Overview

The **Restaurant Management System** is an SQL-based project designed to efficiently manage restaurant operations such as customers, employees, menu items, orders, and billing. The system uses a normalized MySQL database with optimized queries, stored procedures, triggers, and joins to ensure smooth end-to-end process flow.

---

##  Database Schema

The project includes five core tables:

* **Customers** – Customer information
* **Employees** – Staff details and roles
* **Menu** – Menu items, categories, and prices
* **Orders** – Order summary and metadata
* **Order_Details** – Item-wise breakdown of each order

These tables are connected using **foreign keys**, ensuring referential integrity and structured relational mapping.

---

##  Key Features

* Complex **JOIN queries** for order and billing reports
* **Stored procedures** to automate order creation
* **Triggers** to auto-update totals
* **Aggregate functions** for sales analysis
* **Window functions** for performance ranking
* Fully **normalized (3NF)** schema
* Data cleaning, indexing, and optimization implemented

---

##  Outcome

This system simplifies customer handling, order tracking, billing accuracy, and employee performance monitoring. It offers reliable and real-time restaurant insights.
